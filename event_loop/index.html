Event loop

Hi, my name is Nastya. I am junior Front-end developer. Today I will tell you about event loop.
Okay, look at the picture. You can see a scheme of an engine for processing JavaScript. An engine interacts with memory heap and the call stack. Memory heap is where the memory allocation happens. The call stack - is the execution context, a structure, which records where in the program we are. The call stack can contain only one function at the moment. Also there are some functions that are provided by browser API and not described in EcmaScript specification, like timeouts, mouse events and etc. And there is a separate queue for them - call queue or callback queue When we run some script in browser, engine goes through the code line by line and decide what to do with each of them: if a line is native javascript - engine places this line to the call stack, if a line is from web API - engine places it to the web API area, and after processing web API is placing code into call queue. Functions in call queue are waiting, while all native functions from the call stack will be launched and after that every function from call queue will be placed to the call stack one by one. LATER, but not necessary right after NOW

Notice, that when you are using setTimeout / setInterval functions functions inside them are not being executed after 1s, for example. After 1s they will be placed to the call queue. In other words, function inside setTimeout or setInterval is executed “later”, but not necessary right after “now”.

Now I am going to explain the reason of this process. Most of the events that depend on user interaction on a page like “click”, “mouseover” and etc. have more priority, than other functions, because there is a vital necessity to keep our page at a life state.

Keeping our page at a life state

There are also some ways to keep our page in a life state. You should break slow functions to small parts if it is possible. For example change this:

Bad:

								
someLongArray = [];
		
someSlowFunc () {};
	
someLongArray.forEach((item) => {
	someSlowFunc(item);
});

Good:

								
someLongArray = [];
		
someSlowFunc () {};
		
someLongArray.forEach((item) => {
	setTimeout(() => {
		someSlowFunc(item);
	}, 0);
});

Avoid synchronic requests

This is synchronic request that can lead to page freezing, it means that browser can’t render, it can’t run any other code, it’s just stuck. So write requests like that:

Bad: 

							
let response = ajax('https://example.com');

console.log(response);

Good: 

							
ajax('https://example.com', function (response) {
	console.log(response);
});
