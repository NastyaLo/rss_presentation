Event loop

Hi, my name is Nastya. I am junior Front-end developer. Today I will tell you about event loop.
Okay, look at the picture. You can see a scheme of an engine for processing JavaScript. An engine interacts with memory heap and the call stack. Memory heap is where the memory allocation happens. The call stack - is the execution context, a structure, which records where in the program we are. The call stack can contain only one function at the moment. Also there are some functions that are provided by browser API and not described in EcmaScript specification, like timeouts, mouse events and etc. And there is a separate queue for them. When we run some script in browser, engine goes through the code line by line and decide what to do with each of them: if a line is native javascript - engine places this line to the call stack, if a line is from web API - engine places it to the web API area, and after processing web API is placing code into call queue. Functions in call queue are waiting, while all functions from the call stack will be launched and after that every function from call queue will be placed to the call stack one by one.
